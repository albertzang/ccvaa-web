# Bug report

**ID:** BUG-20260710-02  
**Reporter:** CEO (manual)  
**Date:** 2026-07-10  
**Environment:** Production (https://ccvaa-web.vercel.app/)  
**Do not use:** ccvaa.ca (CEO manual only)  
**Severity:** high  
**Status:** open

## Summary

Admin OTP email delivers successfully on Production, but verifying the code fails with “No active code found. Request a new one.”

## Steps to reproduce

1. Open https://ccvaa-web.vercel.app/admin at desktop width.
2. Click **Send login code** — email arrives at `info@ccvaa.ca` with a 6-digit code.
3. Enter the code and submit **Verify and sign in**.
4. Observe error: `No active code found. Request a new one.`

## Expected

Valid, unexpired OTP from the email signs the user in and shows post-login scaffolds.

## Actual

Verify fails with missing-challenge error despite a freshly received code.

## Evidence

- CEO manual test after Vercel Production env + redeploy (SMTP send works).
- UI error matches `src/app/api/admin/otp/verify/route.ts` → `verifyOtpChallenge` `missing` message.

## Suspected area (optional)

In-memory OTP challenge store (`src/lib/admin/otp.ts` global `Map`) is **per serverless instance**. Request may land on instance A; verify on instance B → challenge not found. Same class of issue as documented rate-limit/OTP memory notes in FEATURES.md.

## Suggested owner

Developer — durable shared store for OTP challenges (and likely rate limits) on Vercel, or other fix that works across instances.
