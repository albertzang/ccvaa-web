# Bug report

**ID:** BUG-20260710-01  
**Reporter:** QA  
**Date:** 2026-07-10  
**Environment:** Production (https://ccvaa-web.vercel.app/)  
**Do not use:** ccvaa.ca (CEO manual only)  
**Severity:** high  
**Status:** open

## Summary

Admin OTP request on Production fails because `SMTP_PASS` is not set in the Vercel Production environment, so login codes cannot be emailed.

## Steps to reproduce

1. Open https://ccvaa-web.vercel.app/admin at desktop width (≥ tablet).
2. In **Admin login**, click **Send login code**.
3. Observe the error under the login form.
4. (Optional) `POST https://ccvaa-web.vercel.app/api/admin/otp/request` and inspect the JSON error.

## Expected

A 6-digit OTP is emailed to `info@ccvaa.ca` (or a clear, production-appropriate error if mail is temporarily unavailable). Admin login remains usable on Production.

## Actual

UI shows: `SMTP_PASS is not set. Add your Hover mailbox password to .env.local (and set ADMIN_OTP_DEV_MODE=false).`  
API returns a non-OK response with the same configuration error. No code is delivered. Client cooldown still engages (`Resend in Ns` / “Please wait before requesting another code.”).

## Evidence

- Browser automation against https://ccvaa-web.vercel.app/admin (Chromium / Playwright), desktop 1280×800.
- Error text rendered in the Admin login section after **Send login code**.
- Message incorrectly points operators at `.env.local` (local-dev wording) rather than Vercel Production env vars.
- Related code: `src/lib/admin/email.ts` throws when `SMTP_PASS` is missing.

## Suspected area (optional)

Production Vercel env: missing `SMTP_PASS` (and possibly related `SMTP_*` / `ADMIN_OTP_*`). Error copy in `src/lib/admin/email.ts` / OTP request API.

## Suggested owner

Developer (set/redeploy Production SMTP env) + Product Manager triage for error-copy wording
